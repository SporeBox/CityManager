    AI-ul folosit pentru scrierea codului pentru funcția filter este Claude. Am ales acest asistent virtual din cauza abilităților sale superioare de generare a codului și interpretare a prompt-ului, în comparație cu restul asistenților.
    Prompt folosit: Lucrez la un proiect in C in format UNIX pentru materia Sisteme de Operare care are ca scop crearea unui program in stil city manager. Am un struct numit Report cu urmatoarele valori: int rapID, char inspName[30], float latitude, float longitude, coordonatele astea se afla intr-un struct numit Coord, char issue[30], int sevLvl, un struct Timestamp cu variabile int day/month/year/hour/minute, si char description[200]. Scrie o functie int parse_condition(const char* input, char* field, char* op, char* value) care sa imparta un string de forma field:operator:value in cele 3 parti, si sa returneze 1 pentru succes si 0 pentru failure. Scrie si functia int match_condition(Report* r, const char* field, const char* op, const char* value) care returneaza 1 daca propozitia este adevarata si 0 daca este falsa, aceasta va fi folosita pentru a verifica daca un anumit field din raport satisface conditia data pentru filtrare sau nu. Pentru field se pot folosi campurile: severity, category, inspector, timestamp. Ca si operatori se pot folosi ==, !=, <, <=, >, >=.
    Asistentul a generat codul surprinzător de bine si explicat prin comentarii, astfel nu a fost nevoie de intervenția mea în modificarea codului. Am putut folosi cu succes cele două funcții generate de el pentru funcția de filtrare filter_reports.
    Succestul generării de cod se datorează explicării contextului cerinței si oferirea tuturor detaliilor necesare pentru a face acest lucru. Concluzia trasă după acest exercițiu este că un asistent virtual este capabil să realizeze cerințe specifice, însă trebuie menționate toate detaliile și modul în care cerința trebuie să fie rezolvată.
